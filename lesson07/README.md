# Блок 7. Итоговый проект

## Цель итогового проекта

Продемонстрировать навыки, полученные в ходе курса:

- Развёртывание инфраструктуры (Docker, Docker Compose)
- Сбор и обработка логов (Vector)
- Хранение данных (PostgreSQL, DuckDB, Parquet)
- SQL-аналитика для выявления угроз
- Обработка данных на Python (Polars)
- Визуализация (Grafana)

## Занятие 7.1: Постановка задачи

### Формат работы

- __Основная работа:__ выполняется самостоятельно вне занятий
- __Занятие 7.1:__ постановка задачи, консультации по архитектуре
- __Занятие 7.2:__ защита проектов (5-7 минут на студента)

### Базовый сценарий: "Аналитика инцидента"

Вы — аналитик SOC. Поступил сигнал о возможной компрометации. Ваша задача — провести расследование и подготовить отчёт.

__Исходные данные:__

- Auth-логи (из генератора Блока 1)
- Дополнительные данные на ваш выбор

__Требования к проекту:__

1. __Инфраструктура__
   - Docker Compose с необходимыми сервисами
   - Автоматическая загрузка данных

2. __Аналитика__
   - Минимум 5 SQL-запросов для расследования
   - Использование оконных функций
   - Выявление подозрительной активности

3. __Python-обработка__
   - Jupyter notebook с анализом
   - Feature engineering или ML (опционально)

4. __Визуализация__
   - Дашборд в Grafana (минимум 4 панели)
   - Или визуализация в Python

5. __Отчёт__
   - README с описанием инцидента
   - Timeline событий
   - Выводы и рекомендации

### Альтернативные темы

Можете выбрать свою тему по согласованию с преподавателем:

__1. Мониторинг веб-приложения__

- Парсинг и анализ Nginx/Apache логов
- Выявление атак (SQL injection, XSS, сканирование)
- Дашборд с метриками безопасности

__2. Анализ сетевого трафика__

- Firewall-логи или NetFlow данные
- Обнаружение сканирования портов
- Выявление подозрительных исходящих соединений

__3. DNS-аналитика__

- Анализ DNS-запросов
- DGA-детекция (энтропия, ML)
- Выявление C2-коммуникаций

__4. SIEM-lite__

- Корреляция событий из 3+ источников
- Автоматическое обнаружение инцидентов
- Алертинг в Grafana

__5. Свободная тема__

- Согласуйте с преподавателем
- Должна включать все компоненты курса

### Структура проекта

```
project/
├── README.md           # Описание проекта и выводы
├── docker-compose.yml  # Инфраструктура
├── data/               # Исходные данные (или ссылка)
├── sql/                # SQL-запросы
├── notebooks/          # Jupyter notebooks
└── grafana/            # Дашборды (JSON export)
```

## Занятие 7.2: Защита проектов

### Регламент защиты

__Время:__ 5-7 минут на студента

__Структура выступления:__

1. __Постановка задачи__ (1 мин)
   - Какую проблему решали?
   - Какие данные использовали?

2. __Демонстрация решения__ (3-4 мин)
   - Архитектура инфраструктуры
   - Ключевые запросы/код
   - Дашборд

3. __Результаты и выводы__ (1-2 мин)
   - Что обнаружили?
   - Какие рекомендации?

__После выступления:__ вопросы от преподавателя и других студентов

### Критерии оценки

| Критерий | Баллы |
|----------|-------|
| Работающая инфраструктура | 20 |
| Качество SQL-аналитики | 25 |
| Python-обработка | 20 |
| Визуализация | 15 |
| Отчёт и выводы | 10 |
| Защита проекта | 10 |
| __Итого__ | __100__ |

__Бонусные баллы:__
- Использование ML: +10
- Оригинальная тема: +5
- Автоматизация (CI/CD, скрипты): +5

### Вопросы для обсуждения

1. Почему выбрали именно такую архитектуру?
2. Как бы решение масштабировалось на больший объём данных?
3. Какие ложные срабатывания возможны?
4. Как можно улучшить детекцию?
5. Какие альтернативные подходы рассматривали?
